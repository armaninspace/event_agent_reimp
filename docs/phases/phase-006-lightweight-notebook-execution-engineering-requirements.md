# Phase 006: Lightweight Notebook Execution

## Phase Goal

Stop treating generated notebooks as inert scaffolds by adding a lightweight execution backend that runs simple generated Python cells, captures output, records execution metadata, and tightens the 20-turn regression gate.

## Requirements

- Add lightweight notebook execution support.
- Generated notebooks must include a validation code cell.
- Execute generated code cells without launching a Jupyter kernel.
- Capture stdout as notebook cell outputs.
- Record execution count and execution status.
- Update notebook metadata from `scaffolded` to `lightweight_executed`.
- Update Markdown exports after execution.
- Update 20-turn regression summary to require all generated notebooks to be lightweight-executed.
- Add tests for execution success, output capture, workspace execution, and regression summary checks.

## Non-Goals

- `nbclient` execution.
- Full Jupyter kernel support.
- Arbitrary untrusted notebook execution.
- Statistical validation.
- Business report or playback UI.

## Assumptions

- The executor only runs notebooks generated by this project.
- Generated validation cells are deterministic and do not require external services.
- Lightweight execution is useful demo evidence but is not publishable statistical validation.

## Affected Layers

- Notebook/evidence layer
- Regression validation
- Tests
- Documentation

## Affected Modules

- `app/notebook_workspace.py`

## Dependency/Library Choices

No new dependencies are required. The executor uses Python standard library modules: `ast`, `contextlib`, `io`, `json`, and `pathlib`.

## Architecture Notes

The lightweight executor is intentionally narrow. It executes generated Python code cells in a shared namespace, captures stdout, writes stream outputs back into the notebook JSON, and updates execution metadata. It does not execute arbitrary external notebooks as a security boundary.

## Data/API/Config Changes

- Phase regression now executes notebooks under the phase notebook workspace.
- Regression summary includes executed notebook counts and validation failures.

## Demo Requirements

- Run the 20-turn regression.
- Show all 20 notebooks are lightweight-executed.
- Show validation failures are zero.

## Test Requirements

- Unit tests verify one notebook executes and captures output.
- Unit tests verify workspace execution counts notebooks.
- Unit tests verify regression requires executed notebooks.
- Validation passes.

## Security/Sandbox Considerations

- Do not execute user-provided notebooks.
- Do not make network calls.
- Do not read `.env`.
- Keep generated code cells deterministic and limited.

## Risks

- Lightweight execution can be overclaimed; reports must distinguish it from `nbclient_executed` and `validated`.
- The executor is not a Jupyter replacement.
- Later phases must add `nbclient` for higher-fidelity execution.

## Acceptance Criteria

- Generated notebooks include validation code cells.
- 20-turn regression reports 20 lightweight-executed notebooks.
- Validation failures are zero.
- Tests and validation pass.

## Rollback Plan

Revert the Phase 006 commit or remove notebook execution integration, generated artifacts, tests, and Phase 006 docs.
