# Backlog Slice

Use this skill to implement one coherent backlog item at a time.

## Slice Loop

1. Restate the slice goal.
2. Inspect relevant files before editing.
3. Implement the smallest coherent change that satisfies the slice.
4. Add or update tests appropriate to the risk.
5. Run targeted tests.
6. Run relevant e2e checks.
7. Update docs and backlog status.
8. Record evidence in the phase demo or status document.

## Expectations

Keep each slice small enough to review. Prefer existing project patterns. Avoid unrelated refactors. If the slice changes shared behavior, expand validation accordingly.

## Status Updates

When a slice is complete, mark its backlog status `done` and include the test/evidence command. If validation is not applicable, state why.
