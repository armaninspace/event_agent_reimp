# Demo And Status

Use this skill at the end of each phase.

## Demo Document

Create or update `docs/demos/<phase-id>-demo.md` with:

- setup
- commands
- UI steps, when applicable
- expected behavior
- observed behavior
- screenshots, logs, transcripts, or command output when available
- known gaps
- requirement mapping

For UI, dashboard, explorer, chart, report visualization, or other visual phases, reference an illustrative video under `demos/` or document why video recording/generation was blocked.

For non-UI phases, include either a generated walkthrough video or an explicit non-applicability rationale with command-output evidence.

## Deliverables Status

Create or update `docs/deliverables-status/<phase-id>-status.md` with:

- completed items
- blocked/deferred items
- files changed
- dependencies installed
- tests run
- demo evidence
- video path or video-blocker/non-applicability rationale
- acceptance criteria status
- risks
- next phase
- commit hash, when available
- label/tag, when available
