# Replan On Block

Use this skill when a backlog item or phase cannot proceed as planned.

## Blocker Handling

1. Identify the root cause.
2. Capture evidence: failing command, missing file, unavailable dependency, permission issue, or unclear requirement.
3. Attempt a safe unblock path.
4. If unresolved, mark the item `blocked`.
5. Add a replan section to the phase or backlog document.
6. Continue with unblocked work where possible.

## Replan Section

The replan should include:

- blocker summary
- attempted fixes
- remaining constraints
- revised path
- deferred work
- validation impact

Do not use destructive actions, remote pushes, merges, or publication as unblock paths without explicit permission.
